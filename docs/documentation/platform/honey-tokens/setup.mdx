<Info>
  This setup is performed **once per organization** by an admin. After
  completing these steps, any project member can [create honey
  tokens](/documentation/platform/honey-tokens/usage) without repeating this
  process.
</Info>

## Prerequisites

- Create an [AWS App Connection](/integrations/app-connections/aws) with the following IAM permissions:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "HoneyTokenIAMManagement",
      "Effect": "Allow",
      "Action": [
        "iam:CreateUser",
        "iam:DeleteUser",
        "iam:CreateAccessKey",
        "iam:DeleteAccessKey"
      ],
      "Resource": "arn:aws:iam::*:user/inf_ht_*"
    },
    {
      "Sid": "HoneyTokenStackVerification",
      "Effect": "Allow",
      "Action": ["cloudformation:DescribeStacks"],
      "Resource": "*"
    }
  ]
}
```

<Note>
  The IAM permissions are scoped to users prefixed with `inf_ht_`, which is the
  naming convention Infisical uses for honey token IAM users.
</Note>

## Configure the Organization

<Steps>
  <Step title="Navigate to Organization Settings">
    Go to **Organization Settings** → **Product Settings** and scroll to the **Honey Tokens** section.

    {/* <!-- Screenshot: Organization Settings page showing the Honey Tokens section with empty form fields --> */}
  </Step>
  <Step title="Select an AWS App Connection">
    Choose the AWS App Connection that has the IAM permissions listed in the prerequisites above.

    ![Select AWS App Connection](/images/platform/honey-tokens/1.png)
  </Step>
  <Step title="Deploy the CloudFormation stack">
    The settings page displays a pre-filled AWS CLI command with your webhook URL, signing key, and stack name already populated. Copy the command from the UI and run it in your terminal.

    ![Deploy CloudFormation Stack](/images/platform/honey-tokens/2.png)

    <Tip>
      Always copy the command from the settings page — it includes your actual
      webhook URL and signing key pre-filled.
    </Tip>

    Wait for the stack to reach `CREATE_COMPLETE` status before proceeding. You can check with:

    ```bash
    aws cloudformation describe-stacks \
      --stack-name infisical-honey-tokens \
      --query "Stacks[0].StackStatus"
    ```
  </Step>
  <Step title="Advanced Options (optional)">
    **CloudFormation Stack Name** — defaults to `infisical-honey-tokens`. You can customize this if
    you have naming conventions or need multiple stacks. The name must be 1–128 characters.

    ![Advanced Options](/images/platform/honey-tokens/3.png)
  </Step>
  <Step title="Save the configuration">
    Click **Save**. Infisical verifies that the CloudFormation stack is deployed before saving.

    ![Save configuration](/images/platform/honey-tokens/4.png)
  </Step>
</Steps>

Setup is complete. You can now [create honey tokens](/documentation/platform/honey-tokens/usage) in any project.
