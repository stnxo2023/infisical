---
title: "Honey Tokens"
sidebarTitle: "Overview"
description: "Detect unauthorized access to your secrets by planting decoy AWS credentials monitored by Infisical."
---

Honey tokens are decoy credentials that act as tripwires. When an attacker gains access to your secrets and attempts to use a honey token, Infisical detects the activity and immediately alerts your team.

Each honey token is planted alongside your real secrets so it appears genuine. Any attempt to use these credentials triggers an alert and notifies organization admins.

## Getting Started

<Steps>
  <Step title="One-Time Setup">
    An organization admin clicks **Connect** to configure AWS Honey Tokens, uses **Manage** to update settings, deploys the CloudFormation stack, and verifies with **Test Connection**. This only needs to be done **once per organization**.

    Follow the [Setup Guide](/documentation/platform/honey-tokens/setup).
  </Step>
  <Step title="Create and Manage Tokens">
    Once setup is complete, any project member with the appropriate permissions can create, monitor, and manage honey tokens across projects.

    Follow the [Usage Guide](/documentation/platform/honey-tokens/usage).
  </Step>
</Steps>
