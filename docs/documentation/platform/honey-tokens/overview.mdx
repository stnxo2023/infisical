Honey tokens are decoy credentials that act as tripwires. When an attacker gains access to your secrets and attempts to use a honey token, Infisical detects the activity via AWS CloudTrail and immediately alerts your team.

Each honey token consists of a real but unprivileged AWS IAM user and access key pair. The credentials are planted alongside your real secrets so they appear genuine. Any API call made with these credentials triggers an alert.

## How It Works

1. Infisical creates a no-permission IAM user in your AWS account via an [AWS App Connection](/integrations/app-connections/aws).
2. A CloudFormation stack monitors CloudTrail for any API calls made with that IAM user's access key.
3. When activity is detected, a signed webhook notifies Infisical.
4. Infisical marks the honey token as **Triggered** and sends an email alert to organization admins.

## Getting Started

<Steps>
  <Step title="One-Time Setup">
    An organization admin configures the AWS connection and deploys the CloudFormation stack. This only needs to be done **once per organization**.

    Follow the [Setup Guide](/documentation/platform/honey-tokens/setup).
  </Step>
  <Step title="Create and Manage Tokens">
    Once setup is complete, any project member with the appropriate permissions can create, monitor, and manage honey tokens across projects.

    Follow the [Usage Guide](/documentation/platform/honey-tokens/usage).
  </Step>
</Steps>
